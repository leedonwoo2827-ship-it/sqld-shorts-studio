# 파일명 규정

파일만 봐도 *무슨 문제 · 누가 발표* 인지 알 수 있게 (자기설명적).

## 강사 코드
- **남성 M1~M5 · 여성 F1~F5** (총 10명). 파일명엔 **소문자**(m1~m5 / f1~f5)로.
- 얼굴 코드 = 목소리 코드가 같으면 톤이 매칭됨(예 `f2` 얼굴 ↔ `f2` 음성).

## 문제 JSON (입력)
```
{과목}_{문제번호4자리}_{얼굴}_{목소리}.json      예) sqld_0007_f2_f2.json
```
- **한 파일 = 한 문제** (JSON 안 `blocks` 에 problem 1개). 여러 문제를 한 파일에 담아도(blocks 여러 개) 웹에서 자동 인식.
- **파일명 뒤 `_f2_f2`** = 이 문제를 진행할 강사(얼굴·목소리). 웹에 첨부하면 **자동 선택**됨.
- 과목 접두사(sqld/…)라 앞자리 0 걱정 없음. 과목만 바꾸면 어떤 자격증에도 적용.
- 필드: `question, choices[], answer, answer_index, explanation, narration(낭독 대본, 숫자는 "이 번"처럼 한글), difficulty, tags[]`

## 결과 MP4 (출력)
```
{과목}_{문제번호4자리}_{얼굴}_{목소리}.mp4       예) sqld_0007_f2_f2.mp4
```
- 첨부한 JSON과 **같은 이름 + .mp4**. (웹에서 얼굴·목소리를 바꾸면 그 코드로 반영)

## ⚠️ 문제번호 ≠ 강사코드 (별개)
- `sqld_0007_f2_f2.json` = **문제 7번**(TRUNCATE)을 **F2 강사**가 진행 → `sqld_0007_f2_f2.mp4`.
- 문제번호(0007)는 *무슨 문제*, 강사코드(f2)는 *누가 읽나*. 서로 무관.

## 샘플 (10개, 문제별로 다른 강사 데모)
`sqld_0001_m1_m1.json`(문제1·M1) … `sqld_0005_m5_m5`(M5) · `sqld_0006_f1_f1`(F1) … `sqld_0010_f5_f5`(F5)
